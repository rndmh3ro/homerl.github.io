SAS JBOD issues

Posted on 2018-06-05   |   In storage   |  

issue introduction in supermicro JBOD

debug sas connection from driver layer

I found there are a lot of counts increment in supermicro 4U 90 bay JBOD.
If you see running_disparity_error_count and phy_reset_problem_count increment in your server, that means too bad thing in the sas connection.

/sys/class/sas_phy means in driver layer ,not means sas device.
If you found somethings wrong in driver layer, that means.

Here is driver layer, not means there is the issue in sas devices, SAS HBA, JBOD IO expader, both of which are likely.
And the HDD has a lot of ECC delayed, JBOD vibrate will cause this issue. but not all.

If you want make sure sas dev whether has the issue, you could get it from smartctl

About parameters

  • Invalid DWORDs
    • The number of invalid dwords that have been received, outside of PHY reset sequences, since the last PHY Link Error Reset
  • Loss of DWORD Sync:
    • The number of times, since the last PHY Link Error Reset, that DWORD synchronization was lost and the link reset sequence occurred.
  • Running Disparity Errors
    • The number of DWORDS with running disparity errors that have been received, outside of PHY reset sequences, since the last PHY Link Error Reset
  • PHY Reset Errors
    • The number of times the PHY reset sequence has failed, since the last PHY Link Error Reset
